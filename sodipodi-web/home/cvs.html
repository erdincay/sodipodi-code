The freshest Sodipodi can always be found at Gnome CVS, module sodipodi. Well, strictly speaking the absolute up to date version is in my computer, but I usually keep the main CVS quite synchronized. What, of course, means, that the code in CVS can be totally out of order. If it compiles, it is good, if it runs, it is even better, but do not expect it to do the work!

Still, even if you are occasionally compiling programs yourself, using CVS pays back the little extra work.

  • You can get hands on new and cool features before the rest
  • Bugs are fixed in CVS initially, until they find a way to release
  • You can test new features and give feedback, before releases are done

If you are unfamiliar with CVS, here is simple overview to get started. This is shamelessly copied (and abbreviated) from http://developer.gnome.org/tools/cvs.html, so look there, if you want to get more information.

First set up CVSROOT environment variable: 

$ export CVSROOT=':pserver:anonymous@anoncvs.gnome.org:/cvs/gnome'

Assuming you are running bash. If not, you probably know yourself, what to do. Given CVSROOT points to readonly copy of Gnome CVS.

Then run the command: 

$ cvs login

If asked for password, just press [Return]. You only need to run this command once. All later accesses to the same CVSROOT will do login automatically.

Now check out initial copy of sodipodi module: 

$ cvs -z3 checkout sodipodi

The initial checkout may take some time, as all files have to be downloaded to local computer. Updating local copy later is usually much faster, as CVS download only differences between local copy and server version.

CVS copy does not include all autoconf, automake and gettext files, that are usually in released tarball. Most importantly the configure script has yet to be generated. Just chdir into base directory, and run: 

./autogen.sh [options]

Where [options] are the same options you normally use for configure.

If autogen.sh succeeds, just continue, as usual: 

make
make install

If you want later to update your local copy to the latest version in server, just chdir into source tree root, and run: 

cvs update -d

Thats all. More information and link to general CVS documentation can be found at http://developer.gnome.org/tools/cvs.html.
