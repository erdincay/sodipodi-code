02 Mars 2004

We are trying to set up framework for doing regular CVS builds, so brave people can have hands on new features faster (and we can get feedback faster). Currently one can find tarballs and build logs at http://lauris.kaplinski.com/sodipodi/nightly/. There is also (a bit older) Windows binary (2004/03/01).

New features:

  • You can 'enter the group' from context menu - but only rect tool works for now :-|
  • There is document-tree dialog, that allows you to parse objects and toggle visibility/sensitivity/printability

11 Veebruar 2004

Sodipodi 0.34 released

The fresh new release of sodipodi is out for your enjoyment. Read Release Notes for the overview of new features. The tarball is immediately available, binary packages will follow very soon.

Packages so far:

  • Source tarball
  • Windows 2000/XP installation package (Unicode version) (Lauris)
  • Windows 9X/ME installation package (ANSI version) (Lauris)
  • Mandrake 9.2 (i386) binary and source rpm-s (Lauris)
  • Red Hat 9.0 i686 and i386 binary and source rpm-s (Dag)
  • Fedora Linux i686 and i386 binary and source rpm-s (Dag)
  • Mandrake 9.2 x86/64 (Athlon64/Opteron) binary rpm (Thomas). It includes few patches compared to stock 0.34 for 64bit cleanness.

7 Veebruar 2004

0.34 beta is out

The beta package of the next Sodipodi release is out. Go download it and report back all showstopper bugs.

Available packages

  • Source tarball
  • Windows 2000/XP installation package
  • Windows 9X/ME installation package

You may also try to install Win2000/XP package on 9X/ME system, and report back problems. We would really like to drop multiple builds, but until now have not yet managed to make all-unicode version working on Win9X/ME.
